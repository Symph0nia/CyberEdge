# Security Policy

## Reporting a vulnerability

Do not open a public Issue for a suspected vulnerability. Use GitHub's private vulnerability reporting for this repository. Include the affected revision, impact, reproduction steps, and any suggested mitigation. Do not include real customer targets, credentials, personal data, or unnecessarily destructive proof of concept material.

Maintainers will acknowledge a complete report when capacity permits, investigate it privately, and coordinate disclosure after a fix is available. No response-time or bounty commitment is implied.

## Supported versions

CyberEdge is pre-1.0. Security fixes are made on the current `main` branch. Older revisions are not supported.

## Research boundary

CyberEdge is intended only for authorized defensive security work. Reports and tests must use systems owned by the researcher or targets covered by explicit written authorization.
